Here’s a list of the commands with their short descriptions:

- **git --version**: Check the installed version of Git.
- **clear**: Clear the terminal screen.
- **ls -ltr**: List files in a directory with details, sorted by modification time.
- **which git**: Show the path of the installed Git.
- **whoami**: Display the current logged-in user.
- **mkdir git-tutorial**: Create a new directory named "git-tutorial."
- **cd git-tutorial**: Change into the "git-tutorial" directory.
- **mkdir git-project-batch-8**: Create another directory named "git-project-batch-8."
- **pwd**: Print the current working directory.
- **git init**: Initialize a new Git repository in the current directory.
- **ls -a**: List all files, including hidden files.
- **touch laila.txt**: Create a new file named "laila.txt."
- **git status**: Show the current status of the Git repository.
- **git add laila.txt**: Stage the "laila.txt" file for the next commit.
- **git add majnu.txt**: Stage another file "majnu.txt" for the next commit.
- **git commit -m "Added two new text files"**: Commit the staged files with a message.
- **git config --global user.email "your-email@example.com"**: Set your Git email globally.
- **git config --global user.name "Your Name"**: Set your Git username globally.
- **git log**: Display the commit history of the repository.
- **rm majnu.txt**: Delete the file "majnu.txt."
- **git checkout -b dev**: Create and switch to a new branch called "dev."
- **git checkout master**: Switch back to the "master" branch.
- **touch kadoo.txt**: Create a new file named "kadoo.txt."
- **git push origin master**: Push the local "master" branch changes to the remote repository.
- **history**: Display a history of all commands used in the terminal.

Each of these commands is a step in the typical Git workflow for setting up, managing, and working with a Git repository.
